GraphQL Approach

GraphQL

  • A data query language
  • Developed by Facebook, specification open sourced in 2015
  • Used by Facebook, GitHub, Twitter, Instagram, PayPal, Sky and many others
  • Specification: https://facebook.github.io/graphql

Demo Time

Main Advantages

  • Get exactly the data you need, leading to better performance
  • Understand exactly what fields our clients are using
  • Simplifies deprecating old fields, no need for API versioning (e.g. /api/v1/title, /api/v2/title)
  • Enables many different data sources to be easily stitched together (e.g. APIs, DBs, Flat files)
  • Enables real-time push updates to clients through subscriptions. e.g. after a incremental migration
  • Easier maintenance by avoiding a large number of separate endpoints

Main Disadvantages

  • Still maturing as a technology
  • Caching can be more difficult but libraries available to help (e.g. Apollo)
  • Not familiar to our users, requires the use of a client library (we could provide)
  • Development work required to setup and maintain in conjunction with standard REST APIs
  • Fine grained security can be more challenging
  • Dealing with errors requires client to look at response payload since always get a 200 response
  • Potential performance issues with complex queries

Mutations & Subscriptions


mutation ChangeTitle {
  changeStatus(titleNumber: AGL10368, newStatus: "Closed", date: "2018-01-02" ) {
    titleNumber, status
  }

  setApplicationReference(titleNumber: "AGL10368", newApplicationReference: "ABR33434") {
    titleNumber, status, applicationReference
  }
}

subscription Entries {
  entryChanges(titleNumber: "AGL10368") { entries }
  entryDeleted { entryId }
}
